# 指示

指定されたファイルのユニットテストコードを生成すること。
タスク終了時に生成したテストの内容を以下の列を持つ表形式で報告すること。
- テスト対象の関数
- テストケース名
- テスト内容

# 遵守事項

- `describe()`の第一引数はテスト対象の関数名とすること。
- `it()`の第一引数は`<テスト条件>_<検証する振る舞い>`の形式で記載すること。
  - 例: `同一オブジェクトを比較する場合_trueを返すこと`
- 生成したテストに静的解析エラーが無いことを確認すること。
- 生成したテストを実行し、テストが全て通ることを確認すること。
- 非公開のメソッドをテストする際はその関数を直接テストするのではなく、非公開メソッドを呼び出している公開メソッドに対してテストを行うこと
