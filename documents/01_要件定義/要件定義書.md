# 要件定義書

## 1. はじめに

本ドキュメントは、Chrome拡張機能「Super Mouse Gesture（仮）」の要件を定義するものである。本拡張機能は、ブラウザ操作の効率化を目的として、スーパードラッグ機能およびマウスジェスチャ機能を提供する。

## 2. 背景

Webブラウジングにおいて、検索やタブ操作、ページ遷移などの頻繁に行う操作を、マウスのみで迅速かつ直感的に行いたいというニーズがある。既存の操作方法では複数のステップが必要な動作を、ドラッグ操作やジェスチャ操作によって簡略化し、ユーザー体験を向上させることを目指す。

## 3. 対象ユーザー

*   Chromeブラウザを利用してWebブラウジングを行う一般ユーザー
*   マウス操作によるブラウジング効率化を求めるパワーユーザー

## 4. 機能要件

### 4.1. スーパードラッグ機能

テキストまたはリンクをドラッグ＆ドロップすることで、特定のアクションを実行する機能。
本バージョンでは全方向で同一のアクションとするが、将来的に方向ごとのアクション変更や検索エンジンのカスタマイズが可能な設計とする。

| ID | 機能名 | 詳細 | 備考 |
| :--- | :--- | :--- | :--- |
| SD-01 | テキスト検索 | Webページ上のテキストを選択し、任意の方向（上下左右）へドラッグした際、選択されたキーワードで検索を行う。 | 検索結果は新しいタブで表示する。<br>検索エンジンはブラウザ既定のものを使用する。 |
| SD-02 | リンクを開く | Webページ上のリンクを任意の方向（上下左右）へドラッグした際、そのリンク先を開く。 | リンク先は新しいタブで表示する。 |

### 4.2. マウスジェスチャ機能

右クリックを押しながらマウスを特定の方向に移動させることで、定義されたブラウザ操作を実行する機能。
設定画面は本バージョンでは実装しないが、将来的に設定変更が可能となるよう考慮した設計とする。

| ID | 機能名 | 詳細 | ジェスチャ定義 |
| :--- | :--- | :--- | :--- |
| MG-01 | スクロール操作 | ページを上下にスクロールする。 | 上：上へスクロール<br>下：下へスクロール |
| MG-02 | 履歴操作 | ページの履歴を戻る・進む。 | 左：戻る<br>右：進む |
| MG-03 | タブ切り替え | アクティブなタブを切り替える。 | 上左：前のタブを選択<br>上右：次のタブを選択 |
| MG-04 | タブ閉じる＆移動 | 現在のタブを閉じ、隣接するタブを選択する。 | 下左：このタブを閉じて左のタブを選択<br>下右：このタブを閉じて右のタブを選択 |
| MG-05 | ズーム操作 | ページの表示倍率を変更する。 | 左上：ページ拡大<br>左下：ページ縮小 |
| MG-06 | 新規タブ | 新しいタブを開く。 | 右上：新規タブを開く |
| MG-07 | フルスクリーン | フルスクリーン表示の切り替えを行う。 | 右下：フルスクリーン表示 |
| MG-08 | ページトップ/ボトム | ページの最上部または最下部へ移動する。 | 下上：ページの一番上へ<br>上下：ページの一番下へ |
| MG-09 | 再読み込み | ページを再読み込みする。 | 左右：再読み込み |
| MG-10 | タブ復元 | 閉じたタブを開き直す。 | 右左：タブを復元する |

### 4.3. UI/UX機能

ジェスチャ操作時およびスーパードラッグ操作時の視覚的フィードバック機能。

| ID | 機能名 | 詳細 | マウスジェスチャ | スーパードラッグ | 備考 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| UI-01 | ジェスチャトレイル | マウスジェスチャ入力中（右クリック押下中の移動）、マウスの軌跡を画面上に描画する。 | ○ | | |
| UI-02 | アクション名表示 | マウスジェスチャ入力中またはスーパードラッグ操作中、現在の入力に対応するアクション名を画面上に表示する。 | ○ | ○ | ユーザーがクリックを解除した際、またはドロップした際に何が実行されるかを明示する。スーパードラッグではトレイルは表示しない。 |

## 5. 非機能要件

### 5.1. プラットフォーム

*   Google Chrome (最新版)
*   Manifest V3 対応

### 5.2. 性能要件

*   **応答性**: ジェスチャトレイルの描画やアクションの実行において、ユーザーが遅延を感じないスムーズな動作を実現する。

### 5.3. 保守性・拡張性

*   **設定の拡張性**: 将来的な機能追加（設定画面の実装、ジェスチャ割り当ての変更、検索エンジンの変更など）を見越し、設定値やアクション定義をコード内で容易に変更・拡張できる構造とする。
